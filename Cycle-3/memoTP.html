Instanciation d'une classe

Salle salle1;

Modifier une instance

salle1.capacite = 30;
salle1.salleMachine = true;
salle1.libreService = true;
salle1.numero = 213;

Créer et utiliser un constructeur

class Point {
  double abs;
  double ord;

  Salle() {
      capacite = 30;
      salleMachine = true;
      libreService = false;
      numero = 0;
  }

  Salle(int uneCapacite, boolean estUneSalleMachine,
      boolean estEnLibreService, int unNumero) {
      capacite = uneCapacite;
      salleMachine = estUneSalleMachine;
      libreService = estEnLibreService;
      numero = unNumero;
    }
}

Salle salle1 = new Salle(30, true, true, 301);
Salle salle2 = new Salle();

Valeur vs référence

int a = 3;
int b = a;
a = 4;
// b vaut toujours 3
Salle salle1 = new Salle( 30, true, true,  301);
Salle salle2 = salle1;

salle1.capacite = 31;
// salle2.capacite vaut 31.

De classe ou d'instance ?

  • méthode de classe = méthode concernant la classe en général (déclarée static).
  • méthode d'instance = s'applique sur une instance de la classe (= un objet).

Idem pour les attributs.

Comparaison d'objets

  • a.equals(b) pour comparer deux objets a et b,
  • a == b pour comparer deux variables aet b dont le type est primitif (int, double...),
  • a == null pour savoir si l'objet a est null.
  • tests unitaires : assertNull(a) et assertNotNull(a)

Énumérations

Principe

Une énumération est un type de données particulier, dont les valeurs possibles sont listées explicitement. Par exemple, dans le fichier Choix.java :

enum Choix {
    PIERRE, 
    FEUILLE, 
    CISEAUX;
}

Autrement dit, une variable de type Choix peut prendre une valeur uniquement parmi PIERRE, FEUILLE ou CISEAUX. Par exemple ci-dessous la variable c, de type Choix, prend la valeur CISEAUX :

Choix c = Choix.CISEAUX;

On peut ensuite manipuler cet objet c comme n'importe quel objet.

Choix c = ...;
switch (c) {
    case PIERRE:
        System.out.print("P");
        break;
    case FEUILLE:
        System.out.print("F");
        break;
    case CISEAUX:
        System.out.print("C");
        break;
    default:
        System.out.print("?");
}

Méthodes

Les énumérations peuvent également contenir des méthodes.

Observez, dans le code ci-dessous, plus complet que le précédent :

  • la méthode plusFortQue() est une méthode d'instance. Par exemple, CISEAUX.plusFortQue() renverra FEUILLE.
  • la méthode nbChoix() est une méthode de classe. Ainsi Choix.nbChoix() renverra 3.
/**
 * Choix possible au shifumi.
 */
enum Choix {
    PIERRE, 
    FEUILLE, 
    CISEAUX;

    /**
     * Renvoie l'élément battu par le choix courant.
     * @return l'élément battu par le choix courant 
     */
    Choix plusFortQue() {
        Choix plusFort = CISEAUX;
        switch (this) {
            case PIERRE:
                plusFort = CISEAUX;
                break;
            case FEUILLE:
                plusFort = PIERRE;
                break;
            case CISEAUX:
                plusFort = FEUILLE;
                break;
        }
        return plusFort;
    }

    /**
     * Nombre de choix possibles au shifumi.
     * @return nombre de choix possibles au shifumi
     */
    static int nbChoix() {
        return values().length;
    }
}

On remarque dans nbChoix() l'utilisation de la méthode de classe values(), qui renvoie un tableau avec toutes les valeurs possibles pour cette énumération (dans l'ordre de leur déclaration). Ici, Choix.values() renverra donc le tableau {PIERRE, FEUILLE, CISEAUX}.

En fait, en Java, une énumération est une classe un peu spéciale, dont toutes les instances sont déclarées de manière fixe. Ainsi FEUILLE est une instance de l'énumération Choix. De ce fait, on peut également trouver des attributs et des constructeurs dans les énumérations (nous ne les utiliserons pas ce semestre, mais les plus curieux pourront regarder la doc).

Contrairement aux classes, il est possible de comparer deux éléments d'une énumération avec == :

if (choix == Choix.CISEAUX) { ... }
