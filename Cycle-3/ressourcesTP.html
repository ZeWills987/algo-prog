Morpion

En premier lieu,

  • récupérer ce projet Morpion et le sauvegarder sur votre compte,
  • importer ce fichier zip directement dans NetBeans via le menu File / Import project/ From zip..... Ne pas dézipper ce fichier à la main !

Ce projet contient tout le code permettant de jouer au morpion.

Compilez le code du morpion et testez-le. Vous observerez que le calcul des scores est buggé. Pour l'instant ne corrigez pas ce bug.

Observez la classe Coordonnees. Elle contient :

  • 2 attributs, les entiers ligne et colonne. Ainsi un objet de type Coordonnees stocke un numéro de ligne et un numéro de colonne d'une grille.
  • plusieurs méthodes (fonctions) dont static boolean estDansPlateau(Coordonnees coord, int taille), qui indique si les coordonnées coord sont dans la grille carrée de taille taille x taille. Par exemple (0, 0) est dans la grille de taille 14x14, mais pas (-1, 1) : on retrouve ces cas dans les tests unitaires de cette méthode, dans CoordonneesTest.

Faîtes tourner ces tests unitaires, puis corrigez la méthode estDansPlateau() pour qu'ils passent.

Énumération Direction

Observez l'énumération nommée Direction et ses différentes méthodes.

Suivant une direction

La méthode static Coordonnees suivante(Coordonnees c, Direction d) de la classe Coordonnees permet d'obtenir les coordonnées de la case atteinte lorsqu'on part des coordonnées c et que l'on applique la direction d passée en paramètre.

Par exemple :

Coordonnees avant = new Coordonnees(3, 3);
Coordonnees apres = Coordonnees.suivante(avant, Direction.EST);
System.out.println(apres); // affiche (3, 4)

Exécutez le test unitaire de la fonction suivante() dans CoordonneesTest. Vous remarquerez qu'il ne passe pas.

Dans le cas testé, la fonction suivante() est bonne. C'est le test unitaire qui présente une erreur. Identifiez l'erreur commise dans ce test unitaire et corrigez-le.

Cases voisines

La méthode voisines() de la classe Coordonnees prend en paramètre des coordonnées, la taille de la grille (toujours supposée carrée), et renvoie un tableau contenant les coordonnées des cases voisines.

Codez cette méthode.

L'instruction Arrays.copyOf(voisines, nbVoisines) permet ici de renvoyer une copie du tableau voisines ne contenant que ses nbVoisines premières cases.

Codez des tests unitaires pour la méthode voisines(), dans CoordonneesTest. Vous testerez notamment que :

  • une case dans un coin ne possède que 3 voisines,
  • une case sur un bord mais loin d'un coin possède 5 voisines,
  • une case loin des bords possède 8 voisines.

Vérifiez que ces tests passent.

Nombre de voisins amis

Dans la classe Grille, codez la méthode nbVoisinsAppartenantAuJoueur(). Référez-vous à sa documentation pour en comprendre la définition. Vous pourrez utiliser la méthode contenuCase() pour accéder au contenu d'une case à partir de ses coordonnées.

Dans Test Packages, créez la classe GrilleTest dans morpion et ajoutez-y la méthode ci-dessous. Vérifiez que ces tests passent.

@Test
public void testNbVoisinsAppartenantAuJoueur() {
    int[][] contenu = {{2, 1, 1}, {2, 0, 0}, {2, 0, 0}};
    Grille grille = new Grille(3);
    Grille.changerContenuGrille(grille, contenu);
    assertEquals(2, Grille.nbVoisinsAppartenantAuJoueur(grille, 1, new Coordonnees(1, 1)));
    assertEquals(3, Grille.nbVoisinsAppartenantAuJoueur(grille, 2, new Coordonnees(1, 1)));
    assertEquals(0, Grille.nbVoisinsAppartenantAuJoueur(grille, 1, new Coordonnees(2, 2)));
    assertEquals(1, Grille.nbVoisinsAppartenantAuJoueur(grille, 2, new Coordonnees(0, 0)));
}

Une fois les tests passés, réalisez plusieurs parties et vérifiez que tout est conforme à la vidéo du début.

Distributeur de bonbons

Énumération Parfum

Créez l'énumération Parfum dans un fichier Parfum.java. Cette énumération contient les 6 saveurs définies dans les fonctionnalités.

Classe Bonbon

Attributs

Créez la classe Bonbon dans un fichier Bonbon.java. Cette classe a 4 attributs d'instance et 1 attribut de classe :

  1. un String de nom denomination qui permet de stocker le nom du bonbon ("tigre", ...).
  2. un Parfum de nom parfum qui permet de stocker le parfum du bonbon.
  3. un double de nom poids qui permet de stocker le poids du bonbon.
  4. un double de nom calories qui permet de stocker le nombre de calories contenues dans le bonbon.
  5. un attribut de classe de type int permettant de savoir combien de bonbons ont été créés.

Constructeur

Créez le constructeur Bonbon(String uneDenomination, Parfum unParfum, double unPoids, double desCalories) qui initialise chaque attribut d'une instance de Bonbon et qui incrémente le nombre de Bonbon créés.

Créez 6 tests unitaires. Ci-dessous un exemple.

Bonbon b = new Bonbon("Chypresers", Parfum.RIZ, 55, 242);
assertEquals(Parfum.RIZ, b.parfum);
assertEquals("Chypresers", b.denomination);
assertEquals(55.0, b.poids, 0.01);
assertEquals(242.0, b.calories, 0.01);

Fabrique de bonbons

Afin de se simplifier l'écriture d'une construction d'un bonbon en particulier, créez les méthodes de classe de création de chaque bonbon possible. Ci-dessous un exemple.

/**
* Méthode statique permettant de construire et de retourner un "chypresers"
*
* @return un bonbon ayant les caractéristique d'un chypresers
*/
static Bonbon chypresers() {
    return new Bonbon("Chypresers", Parfum.RIZ, 55, 242);
}

Ajoutez des tests pour ces nouvelles méthodes, comme par exemple :

Bonbon b = Bonbon.chypresers();
assertEquals(Parfum.RIZ, b.parfum);
assertEquals("Chypresers", b.denomination);
assertEquals(55.0, b.poids, 0.01);
assertEquals(242.0, b.calories, 0.01);

Méthode initiale()

Créér la méthode d'instance char initiale() qui retourne une lettre correspondant au bonbon. 'c' pour caramel, 'C' pour chocolat, 'F' pour fraise, 'N' pour noisette, ...

Écrire les tests unitaires correspondants.

Méthode afficher()

Créer la méthode d'instance afficher() qui affiche les caractéristiques d'un bonbon comme décrit dans la partie Descriptif.

Classe Distributeur

Attributs

Cette classe possède 1 attribut de classe et 2 attributs d'instance :

  1. un attribut de classe de type int dont le nom est CONTENANCE_MAX et dont la valeur contante (50) correspond au nombre maximum de bonbons que peut contenir ce distributeur.
  2. un attribut d'instance de type int de nom nombreBonbons qui permet de stocker le nombre de bonbons dans ce distributeur.
  3. un attribut d'instance de type Bonbon[] de nom contenu qui permet de stocker des Bonbon dans un tableau.

Constructeur

Créer le constructeur Distributeur() qui permet d'instancier le tableau de Bonbon et d'initialiser le nombre de Bonbon à l'intérieur.

Le test unitaire suivant doit passer :

Distributeur d = new Distributeur();
assertEquals(0, d.nombreBonbons);
assertNotEquals(null, d.contenu);
assertEquals(Distributeur.CONTENANCE_MAX, d.contenu.length);

Méthode ajouterBonbon()

Créer la méthode d'instance boolean ajouterBonbon(Bonbon b) qui permet d'ajouter un bonbon dans le distributeur. Cette méthode retourne true si le bonbon a pu être ajouté, false sinon .

Créer 4 tests unitaires. Vous pouvez temporairement diminuer la taille du distributeur pour tester certains cas limites.

Méthode afficherDistributeur()

Créer la méthode d'instance void afficherDistributeur() qui affiche en mode texte le contenu du distributeur (cf partie Descriptif).

Méthode retirerBonbon()

Créer une méthode d'instance permettant de supprimer le dernier bonbon ajouté dans le distributeur. Créer 4 tests unitaires. Attention aux cas limites.

Méthode retirerBonbons()

Créer une méthode d'instance permettant de supprimer plusieurs bonbons en respectant les consignes données dans la partie Descriptif. Créer 4 tests unitaires.

Utilisation des classes créées

Codez les méthodes permettant comme sur la vidéo :

  • d'afficher le menu principal et demander à l'utilisateur l'opération à réaliser ;
